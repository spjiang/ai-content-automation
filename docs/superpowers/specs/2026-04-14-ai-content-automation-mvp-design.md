# AI 自动化内容生成系统 MVP 设计

## 1. 背景与目标

本项目目标是构建一个从热点抓取到图文产出的自动化系统，先以人工发布闭环验证业务价值，再逐步扩展自动发布能力。

### 已确认约束

- 优先路径：热点抓取 -> DeepSeek 图文生成 -> 人工发布
- 热点来源：微博热搜、知乎热榜、抖音热榜（MVP 即三源并行）
- 数据获取：以自研抓取为主，接受选择器维护与反爬带来的持续成本
- 生产策略：规则过滤后自动进入生成队列
- 内容形态：一稿多投，同时产出小红书与抖音图文版本
- 发布策略：MVP 阶段先导出可发布包，不做自动发布
- 指标优先级：内容质量(B) > 业务效果(C) > 稳定性(D) > 生产效率(A)

## 2. 总体架构

采用“单体后台 + 插件化抓取器 + 异步任务队列 + 审核后台”的实现方式。

### 核心模块

- `collector`：三源抓取（微博/知乎/抖音），统一输出标准化话题
- `normalizer`：字段清洗、话题归一、去重、热度评分与排序
- `rule-engine`：黑白名单、关键词、敏感词、热度阈值，命中后自动入生成队列
- `generator`：调用 DeepSeek，生成双平台稿件（抖音图文版、小红书版）
- `packager`：导出人工发布所需内容包（文本 + 素材建议）
- `review-console`：人工审核、驳回、版本追踪
- `observability`：链路监控、告警、指标看板

### 主链路数据流

抓取 -> 标准化 -> 去重排序 -> 规则筛选 -> 生成 -> 审核 -> 导出发布包

### 架构原则

- 抓取器插件化，便于扩展新平台
- 主链路全程留痕（任务状态机 + 审计日志）
- 自动化优先，同时保留人工审核闸门保障质量

## 3. 数据模型与状态机

### 核心实体

- `topic_raw`：原始抓取记录（平台、标题、热度、链接、抓取时间、原始 payload）
- `topic_canonical`：归一化话题（标准标题、聚类键、综合热度、来源集合、去重指纹）
- `content_job`：生成任务（规则版本、提示词版本、输入话题、状态、失败原因）
- `content_asset`：生成内容资产（按平台版本保存标题、正文、标签、封面文案、配图建议）
- `review_record`：审核记录（审核人、结论、驳回原因、修改意见、版本）
- `publish_package`：发布包（结构化 JSON/文档、下载地址、生成时间、有效期）

### 任务状态机

`NEW` -> `QUEUED` -> `GENERATING` -> `GENERATED` -> `IN_REVIEW` -> `APPROVED` -> `PACKAGED`

补充状态：

- `FAILED`：任意阶段失败，附错误码与可重试标记
- `REVISE_REQUIRED`：审核驳回后待修订，可重新入队生成并保留版本链

### 去重与排序策略

- 去重：标题语义相似度 + 关键词归一 + 时间窗口组合成指纹
- 排序：平台热度归一分 + 新鲜度衰减 + 业务偏好权重
- 输出：按配置 TopN 入规则引擎，并支持主题配额减少同质化

## 4. 异常处理与降级

- 抓取失败：按来源维度指数退避重试；连续失败触发告警；单源故障不阻塞全链路
- 生成失败：记录错误分类（超时、限流、内容安全、模板异常），支持自动/手动重试
- 规则问题：保留规则命中明细，支持回放排查误杀与漏放
- 打包失败：资产不丢失，可重新打包；发布包带版本号避免覆盖

## 5. 指标体系（按优先级）

### B. 内容质量（第一优先）

- 审核通过率
- 一次通过率
- 平均修改次数

### C. 业务效果

- 预留人工回填字段（曝光、播放、互动）
- 计算平台维度 CTR/互动率

### D. 稳定性

- 抓取成功率
- 生成成功率
- 任务失败率
- 告警恢复时长

### A. 生产效率

- 热点入库到发布包导出的全链路时长（P50/P90）

## 6. 测试策略

- 质量回归：构建审核样本集，追踪一次通过率趋势
- 效果回填：验证发布回填字段完整性与统计口径一致性
- 稳定性测试：抓取与生成链路集成测试 + 定时健康检查
- 状态机单测：关键状态迁移与失败重试分支覆盖
- 性能观测：链路各阶段埋点，持续输出 P50/P90

## 7. MVP 里程碑

- `M1`：三源抓取 + 归一去重 + 规则引擎 + 队列（打通数据入口）
- `M2`：DeepSeek 双平台一稿多投 + 审核台 + 发布包导出（形成完整业务闭环）
- `M3`：监控告警 + 指标看板 + 重试/回放工具（提升可运营性）

## 8. 范围边界（MVP 不做）

- 平台自动发布（抖音/小红书 API 接入）
- 短视频自动生成（TTS + 混剪）
- 多租户权限体系与复杂结算能力

## 9. 风险与应对

- 抓取稳定性风险：通过插件隔离、重试、告警、可替换抓取策略降低影响
- 内容合规风险：规则引擎 + 审核闸门 + 全量审计日志
- 一稿多投质量风险：平台模板分层（共用中间稿 + 平台适配层）与样本回归

## 10. 后续计划入口

设计确认后，进入实现计划拆解，按 M1 -> M2 -> M3 顺序执行，并在每个阶段设置可验收产出与回归检查。
